# SOULA Deck — скилл для сборки презентаций

Превращает текст слайдов (markdown) в готовую презентацию PowerPoint (`.pptx`)
в фирменном стиле SOULA. Пишешь текст — получаешь оформленные слайды со шрифтом,
цветами, орбом и макетами из нашего шаблона. Дизайн трогать не нужно.

## Как пользоваться (для всех)

1. Напиши текст будущих слайдов в обычном файле `.md` (или прямо в чате).
   Пример — в [examples/example-content.md](examples/example-content.md).
2. Скажи Клоду:
   > **«Сделай презентацию из этого файла в стиле SOULA»** (`/soula-deck`)
3. Клод разложит текст по фирменным макетам и выдаст готовый `.pptx`.
   Открой его в PowerPoint или Keynote.

Это всё. Не нужно знать макеты наизусть — Клод подберёт их сам. Но если хочешь
контроль, можно указать макет для каждого слайда (см. пример, «Способ 2»).

### Маленькие правила, которые делают результат красивым
- **Одна мысль на слайд.** Если в блоке две идеи — будет два слайда.
- **Акцент в заголовке:** одно слово в `*звёздочках*` станет жирным розовым.
- **Коротко и спокойно** — это часть бренда. Воздух важнее, чем плотный текст.
- В макетах со скриншотом телефона место под картинку («ADD SCREEN») вставляешь
  вручную в PowerPoint — текст вокруг Клод уже расставит.

## Установка (для коллеги)

Нужен установленный Python 3 (на macOS уже есть). Внешних библиотек **не нужно**.

**Способ 1 — через GitHub (рекомендуется):**
```bash
git clone https://github.com/zaulichny/soula-deck ~/.claude/skills/soula-deck
```
Чтобы потом получать обновления — `cd ~/.claude/skills/soula-deck && git pull`.

**Способ 2 — вручную:** скачай папку с GitHub кнопкой **Code → Download ZIP**,
распакуй так, чтобы получился путь `~/.claude/skills/soula-deck/SKILL.md`.

После установки в Claude Code напиши `/soula-deck` или просто попроси
«сделай презентацию в стиле SOULA».

## Что внутри (для технических)

| Файл | Зачем |
|------|-------|
| `SKILL.md` | Инструкция, по которой Клод собирает презентацию |
| `assets/template.pptx` | Фирменный шаблон-источник (13 макетов) |
| `layouts.json` | Карта: какой слайд-донор и какие поля у каждого макета |
| `LAYOUTS.md` | Тот же список полей, но для людей, с примерами |
| `scripts/build_deck.py` | Движок: клонирует слайд-донор и подставляет текст |
| `scripts/generate_layouts.py` | Перегенерирует `layouts.json`, если поменяли шаблон |
| `examples/` | Пример входного текста и собранная демо-презентация |

**Принцип работы.** `.pptx` — это zip-архив с XML внутри. Скрипт берёт из шаблона
готовый слайд нужного макета, меняет в нём только текст (сохраняя формат каждого
фрагмента) и пересобирает архив. Поэтому фирменный дизайн воспроизводится точно,
а не «рисуется заново». Заметки докладчика в итоговый файл не переносятся.

**Если обновили шаблон** (`assets/template.pptx`): запусти
`python3 scripts/generate_layouts.py`. Если число текстовых блоков в макетах
изменилось, скрипт сообщит об этом — карту полей нужно будет поправить.
