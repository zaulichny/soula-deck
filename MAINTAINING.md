# Как обновлять скилл SOULA Deck

Этот файл — про то, как менять **шаблон презентации** и публиковать обновление.
Есть два пути: простой (для всех) и подробный (для технических).

---

## 🟢 Простой путь — просто попроси Клода

Ты не обязан разбираться в скриптах. Самый надёжный способ:

1. Положи **новый шаблон** на место старого:
   `~/.claude/skills/soula-deck/assets/template.pptx`
   (имя файла должно остаться `template.pptx`).
2. Открой Claude Code **в папке скилла** и скажи примерно так:
   > «Я заменил шаблон в assets/template.pptx. Обнови скилл под новый шаблон и выложи на GitHub.»
3. Клод сам пересоберёт карту полей, проверит, что всё открывается, и запушит обновление.
4. Коллеги получат новую версию командой `git pull` (или повторным `git clone`).

Если в новом шаблоне **добавились/убрались слайды или блоки текста** — Клоду нужно будет
поправить карту полей. Он это умеет: просто скажи ему, что изменилось
(«добавил макет с таблицей», «убрал слайд про цифры» и т.п.).

---

## 🔧 Подробный путь — шаги вручную

Весь процесс воспроизводимый, зависимостей нет (только Python 3).

### 1. Заменить шаблон
Положи новый файл в `assets/template.pptx`. Дизайн, шрифты, цвета, орб — всё берётся
отсюда. **Правило шаблона:** все слайды должны использовать один общий макет
(`slideLayout1`) и быть «донорами» — по одному слайду на каждый тип макета.

### 2. Перегенерировать карту полей
```bash
python3 scripts/generate_layouts.py     # пересобирает layouts.json
python3 scripts/generate_layouts_md.py  # пересобирает LAYOUTS.md (для людей)
```

- `generate_layouts.py` сверяет шаблон с ожидаемой структурой. Если число текстовых
  блоков в каком-то макете **изменилось**, он упадёт с понятной ошибкой, например:
  `title: форм в шаблоне 12, в карте 11`.
- В этом случае открой `scripts/generate_layouts.py` и поправь словарь `LAYOUTS`:
  там для каждого макета перечислены имена полей **в порядке текстовых блоков слайда**.
  `None` — блок не трогаем (например, нижний колонтитул «SOULA»), `"@n"` — номер
  страницы (проставляется сам). Чтобы увидеть фактический порядок блоков нового
  шаблона, можно попросить Клода «разобрать структуру слайдов шаблона».

### 3. Проверить, что всё собирается и открывается
```bash
python3 scripts/build_deck.py --spec examples/example-spec.json --out /tmp/check.pptx
```
Файл должен собраться без ошибок. Для строгой проверки целостности (что PowerPoint
не предложит «восстановить») открой результат любым валидатором pptx — например,
библиотекой `python-pptx` во временном окружении. Открой `/tmp/check.pptx` глазами
и сравни со старым стилем.

### 4. Обновить демо-пример (необязательно)
```bash
python3 scripts/build_deck.py --spec examples/example-spec.json --out examples/example-deck.pptx
```

### 5. Опубликовать на GitHub
```bash
git add -A
git commit -m "Обновлён шаблон презентации"
git push
```
Коллеги подтянут обновление: `cd ~/.claude/skills/soula-deck && git pull`.

## Что где лежит

| Файл | Менять руками? | Зачем |
|------|----------------|-------|
| `assets/template.pptx` | да — это и есть шаблон | Источник дизайна |
| `scripts/generate_layouts.py` | да, если поменялась структура макетов | Описывает, какие поля у каждого макета |
| `layouts.json` | нет, генерируется | Карта полей для движка |
| `LAYOUTS.md` | нет, генерируется | Справочник полей для людей |
| `scripts/build_deck.py` | редко (только при новых типах полей) | Движок сборки |
| `examples/` | по желанию | Пример входа и демо-преза |

---

## Частые ситуации

- **Поменял только тексты-«рыбу» или цвета внутри существующих макетов** (число блоков
  не изменилось) → достаточно заменить `template.pptx` и прогнать шаг 2. Карту полей
  править не нужно.
- **Добавил новый тип слайда** (новый макет) → добавь его в словарь `LAYOUTS` в
  `generate_layouts.py` (новый ключ с именами полей), затем шаг 2–5.
- **Сломалось после обновления** → почти всегда дело в том, что порядок/число текстовых
  блоков в шаблоне разошлись с картой в `generate_layouts.py`. Скрипт подскажет, какой
  макет не сошёлся.
