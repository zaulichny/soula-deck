# SOULA Deck Builder

Собирает `.pptx` в фирменном стиле SOULA из текста слайдов в markdown.

**Принцип:** дизайн не перерисовывается. Скрипт берёт готовый слайд-донор нужного
макета из шаблона и подставляет в него текст — поэтому шрифт (Arial), цвета
(чёрный `0F0F0F`, розовый акцент `FF2E6E`, кремовый фон `FAF7F3`), отступы, орб и
градиенты сохраняются точно. Менять можно только текст.

## Что нужно от пользователя

Текст слайдов — либо файл `.md`, либо прямо в сообщении. Два режима (поддерживай оба):

- **Авто** — обычный markdown (заголовки + абзацы). Сам подбираешь макет под каждый блок.
- **Явный** — пользователь указал макет: строка `layout: <имя>` в начале блока слайда.
  Если макет указан явно — **уважай его выбор**, не переопределяй.

## Рабочий процесс

1. **Прочитай контент.** Если это файл — прочитай его. Если непонятно, что считать
   отдельным слайдом, — раздели по заголовкам/смысловым блокам и **покажи пользователю
   план слайдов (макет + о чём)** до сборки, если слайдов больше ~5 или есть неоднозначность.
2. **Подбери макет** для каждого слайда (см. каталог ниже). Один слайд = одна мысль.
3. **Перепиши текст в фирменном голосе** (см. правила голоса). Это важно: не копируй
   сырой текст дословно — ужми до спокойных коротких формулировок и расставь акценты.
4. **Собери `spec.json`** (формат ниже) во временной папке.
5. **Запусти сборку:**
   ```bash
   python3 ~/.claude/skills/soula-deck/scripts/build_deck.py --spec <spec.json> --out <имя>.pptx
   ```
   Зависимостей нет, нужен только Python 3. Скрипт сам найдёт шаблон и `layouts.json`
   внутри папки скилла.
6. **Проверь результат** и отдай путь к `.pptx`. Предложи поправки. Если в презентации
   есть макеты со скриншотом (`*-screen`), напомни: место под скриншот телефона
   («ADD SCREEN») заполняется вручную в PowerPoint — текст для него мы расставили.

## Правила фирменного голоса (соблюдай при переписывании текста)

- **Одна мысль на слайд.** Если в блоке две идеи — сделай два слайда.
- **Один акцент на заголовок.** Ровно одно слово или короткую фразу в заголовке
  оберни звёздочками: `Заголовок, который *чувствует* тебя`. Это слово станет жирным
  розовым. Не больше одного акцента. Без акцента — заголовок просто чёрный.
- **Спокойно и коротко.** Лид-абзац — 1–3 предложения. Подписи — одна строка.
  Воздух — часть бренда; не заполняй всё.
- **Параллельная структура** в колонках и в «было/стало»: одинаковая форма и длина строк.
- **Не превышай вместимость макета** (лимиты в каталоге). Если данных больше — раздели слайд.
- Язык вывода = язык контента пользователя (можно и русский, и английский).
  Метки (`PRESENTER`, `BEFORE`, `SLACK` и т.п.) при необходимости переводи через
  соответствующие поля `*_label` / `*_tag`.

## Каталог макетов

Полный список полей каждого макета — в [LAYOUTS.md](LAYOUTS.md) и в `layouts.json`
(машинный источник). Краткий справочник «когда что брать»:

| layout | Когда использовать | Ключевые поля |
|--------|--------------------|----------------|
| `title` | Первый слайд | eyebrow, headline, subtitle, presenter, team, date |
| `section` | Разделитель главы | number, eyebrow, headline, subtitle |
| `context` | Один тезис + пояснение | eyebrow, headline, lead, supporting, note |
| `two-columns` | Сравнение A/B, сейчас/потом | headline, colA_heading/body, colB_heading/body |
| `agenda` | Список/повестка (до 4) | headline, item1..4_heading/body |
| `numbers` | 3 метрики/цифры | headline, intro, stat1..3_value/unit/label |
| `before-after` | Было/стало (по 3 строки) | headline, before_/after_heading, before_/after_lines |
| `roadmap` | 4 фазы/этапа | headline, phase1..4_tag/heading/body |
| `overview` | Тезис слева + до 4 пунктов | headline, intro, bullet1..4_heading/body |
| `steps-screen` | 3 шага + скриншот телефона | eyebrow, headline, step1..3_num/heading/body |
| `features-screen` | 3 пункта пользы + скриншот | eyebrow, headline, feature1..3_heading/body |
| `steps-alt` | 3 шага (компактно) + скриншот | headline, step1..3_tag/heading/body |
| `closing` | Спасибо + контакты | headline, subtitle, contact1..3_label/value |

Лимиты: `agenda`/`roadmap`/`overview` — максимум 4 пункта; `numbers`/`*-screen` — 3.
Лишние пункты можно очистить, передав пустую строку `""` в их поля.

## Формат spec.json

```json
{
  "slides": [
    {"layout": "title", "fields": {
      "eyebrow": "КОМАНДНЫЙ СИНК · ИЮНЬ 2026",
      "headline": "Заголовок, который *чувствует* тебя",
      "subtitle": "Одно спокойное предложение про эту презентацию.",
      "presenter": "Имя", "team": "Product · Soula", "date": "1 июня 2026"
    }},
    {"layout": "two-columns", "fields": {
      "headline": "Два пункта *рядом*",
      "colA_heading": "Сейчас", "colA_body": "…",
      "colB_heading": "Дальше", "colB_body": "…"
    }}
  ]
}
```

- Заголовок: `*слово*` → жирный розовый акцент. Ровно один на заголовок.
- Многострочные поля (`before_lines`, `after_lines`): разделяй строки `\n`.
- Не переданные поля остаются как в шаблоне (английская «рыба»). Поэтому **заполняй
  все видимые поля выбранного макета** или передавай `""`, чтобы очистить.
- Номера страниц проставляются автоматически (01, 02, …) — их задавать не нужно.

## Явный формат для пользователя (режим «явный»)

Если пользователь пишет блоки слайдов сам, ожидай примерно такой markdown и переводи
его в `spec.json`:

```
## layout: two-columns
headline: Два пункта *рядом*
colA_heading: Сейчас
colA_body: …
colB_heading: Дальше
colB_body: …
```

Пример полного входного файла — в [examples/example-content.md](examples/example-content.md).

## Если поменялся сам шаблон

Если заменили `assets/template.pptx` — перегенерируй карту полей:
`python3 scripts/generate_layouts.py`. Проверь, что число форм совпало (скрипт
сообщит об ошибке, если структура макетов изменилась).
